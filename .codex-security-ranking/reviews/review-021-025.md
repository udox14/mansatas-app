## Exact suppressions and negative controls

1. **Tahfidz class lookup only suppressed.** `getKelasTahfidz` at lines 10-22 exposes class identifiers and labels only. In isolation this is low-sensitivity lookup metadata and no concrete security impact beyond the stronger TAH children was found. It does not suppress downstream student/report actions.
2. **TPG action-local access otherwise suppresses.** Upload authenticates at lines 22-23, checks the `tpg-dokumen` feature at 37-39, derives roles from the authenticated user at 40-43, and hard-binds the upload key/database owner to `user.id` at 46 and 49-61. Data listing authenticates and checks the feature at 71-76; non-managers are forcibly reduced to `u.id = ?` at 91-95, while only `super_admin`/`admin_tu` can list others at 77-78. Period inputs are bounded at 11-14. The private-file candidate is retained because these controls are bypassed by the generic media proxy after upload.
3. **Portal-parent object isolation suppresses all assigned exports.** `requireParentSession` rejects non-parent sessions at 65-69, and every export calls it. Payment submission derives the student exclusively from `session.user.siswa_id` at 139-158. Proof upload selects and updates by both attacker submission id and session student at 174-190 before uploading under that submission id. Grade reads bind the session student at 203-208. Summons reads/updates bind both summon and session student at 233-247; notification updates do likewise at 267-271. Suggestions, WhatsApp updates, and password changes bind the session's student/NISN rather than caller ids at 296-307, 338-343, and 372-385. Semester column interpolation is fixed to the six-entry allowlist at 13 and 198-200. SQL values are bound.
4. **Portal payment amount exceeding the remaining balance suppressed as workflow validation, not security.** The parent can submit an amount greater than `sisa`, but the row remains `belum_upload`/`menunggu_konfirmasi`; this file cannot confirm it or create a transaction. No unauthorized object or privileged sink is reached. Candidate should be reopened only if the admin confirmation path trusts `jumlah` without bounding it against the DSPT balance.
5. **Portal payment-proof capability URL suppressed in this file.** Ownership is enforced before upload and the key contains a server-generated UUID submission id. Although the generic media route is public, this assigned action does not disclose another family's URL or permit choosing another family's key. Unlike deterministic TPG keys, no practical enumeration path is evidenced here. Treat URL leakage/caching as a media-route family question, not a parent-action IDOR.
6. **No SQL injection candidate.** All attacker data is bound. TKA's `${col}` is selected solely from `pilihan === 1`, and the parent grade column comes from a fixed tuple. Dynamic placeholder counts are code-generated.

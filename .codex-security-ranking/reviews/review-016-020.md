#### `KD-001` — unauthenticated student search discloses identifiers and photos

- Instance key: `missing-auth-read:app/dashboard/kedisiplinan/actions.ts:312`
- Affected locations: `actions.ts:312-328` (entrypoint/root control/query sink), 330-338 (serialization sink).
- Source/control/sink: unauthenticated two-character `query`; there is no `getCurrentUser` or feature check; `%query%` is bound into a school-wide active-student lookup returning id, name, NISN, photo URL, and class.
- Impact/CWE: bulk-enumerable student PII, CWE-306/CWE-200.
- Candidate-local validation / attack path: invoke the deployed action without cookies using common name prefixes or NISN fragments and inspect the 20-row payload. Internet client -> server-action reference -> D1 search -> PII JSON. SQL injection is not claimed because values are bound at 327.
- Preliminary disposition: reportable if deployed action reachability is confirmed.

- Instance key: `cross-period-object-authz:app/dashboard/kelas-binaan/actions.ts:125`

Proof gap: whether unrestricted retrospective correction is an intended business capability; the repository threat model says historical relationships must not grant current access, and the inverse current-to-history mutation also threatens record integrity.

## Exact suppressions and negative controls

1. **Parameterized SQL / generated placeholders:** no SQL-injection candidate is emitted. Caller values use D1 `.bind`; dynamic placeholder strings in `attendance-actions.ts:56` and `surat/actions.ts:223` are generated solely from array length, and `PAGE_SIZE` at kedisiplinan 764/778 is fixed. The security defects are authorization and object scope.
2. **Discipline delete ownership:** `hapusPelanggaran` loads `diinput_oleh` and permits only owner or super-admin at 453-460. Cross-owner deletion is suppressed. The feature/CRUD-revocation mismatch remains separately tracked as `KD-006J`.
3. **Master/sanction/config role escalation:** manager mutations require one of `super_admin`, `wakamad`, or `guru_bk` at the action boundary. No arbitrary-role escalation was found. Their remaining issue is whether explicit feature/CRUD revocation is meant to override role membership (`KD-006B-I`).
4. **Import row existence checks:** discipline commit checks selected student/master IDs exist at 699-703 and uses the active-year ID at 633-634/733. SQL/object-existence bypass is suppressed. It intentionally includes inactive students and remains a policy-sensitive feature/CRUD candidate, not an injection finding.
5. **Kelas-binaan cross-class summons:** create/cancel authenticate and bind non-admin access to the student's current class/wali at 58-69, 89-90, and 138-139. Caller-controlled `kelas_id` is used only for cache revalidation. Direct summons for another current class are suppressed. `KB-001` concerns the different unscoped historical/creator record selected after that valid current-class check.
6. **Kelas-binaan attendance cross-class payload:** batch action authorizes the class and proves every unique student ID is active in that class at 48-66. Cross-class student injection is suppressed. D1 CHECK constraints from migration 0046 constrain runtime status values; the retained issue is arbitrary historical date scope.
7. **R2 upload type/size:** new discipline evidence calls `validateImageFile` at 378-381 and the shared helper caps allowed images at 2 MB (`utils/r2.ts:7-18`). Unsafe upload type/size is suppressed. `KD-003` is deletion of a different caller-selected key before record authorization.
8. **Act-as cookie path:** `getEffectiveUser` itself requires a real super-admin and validates the target exists (`lib/act-as.ts:50-75`). Cookie-based impersonation is suppressed. `KH-001` is the separate explicit function argument that bypasses this helper.
9. **`getSanksiList` / `getKedisiplinanConfig`:** unauthenticated reads expose only school-wide threshold labels/numbers. `getSanksiList` may create/default the table when empty, but the write is fixed/idempotent and no plausible confidentiality or privilege impact was established. Suppressed as low-impact initialization/config disclosure.

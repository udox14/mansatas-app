## Exact suppressions and negative controls

1. **Agenda-class reads/exports suppressed.** Every exported action that reaches student/attendance data calls `ensureAccess` directly or delegates to an exported function that does (`actions.ts:264-270`, 452-455, 458-461, 479-484, 487-495, 520-525). The default menu grants this explicitly school-wide review/print feature only to `super_admin` and `admin_tu` (`config/menu.ts:191-196`), and the action output is intentionally cross-class. Active teaching data is joined to the active year at 320/383 and 528/676, so old `penugasan_mengajar` rows do not grant access or contaminate schedules. Arbitrary class ids are therefore within the documented school-wide feature scope, not an IDOR under the current model. Date/month formats are constrained at 83-85 and 142-147; SQL values are bound. No mutation sink exists.
2. **Remaining guru actions suppressed.** `editPegawai`, `importPegawaiMassal`, and `uploadFotoPegawaiAction` call `verifyStaffAdminAccess` before their sinks (116, 315, 430). The image path uses `validateImageFile` at 434-435 before R2 upload. These remain privileged admin operations; no non-admin path was found. `setUserRoles` input allowlisting is not suppressed as a mere validation issue because GUR-003 preserves the concrete privilege escalation.
3. **Remaining izin actions suppressed.** Reason-list read at 59-64 is non-sensitive lookup data. Reason mutations require `super_admin` at 68-72 and 92-96. Class lists and student/leave loads authenticate and enforce wali ownership at 123-148 and 152-201. Global operational mutations authenticate and require explicit input roles at 305-310, 330-335, 353-361, 428-433, 477-482, and 522-530; deletions require `super_admin/admin_tu` at 389-407, 460-465, 509-514, and 562-567. The threat model and page UI explicitly give these operational roles school-wide leave handling, so arbitrary leave ids are within that role scope. SQL is parameterized, and edit timestamps/reasons are normalized/allowlisted. IZN-002 is not covered by this suppression because a wali-class action has a narrower object boundary and fails to bind payload student ids to it.
4. **Academic dynamic identifiers are not SQL injection candidates.** `targetKolom` is checked against `SEMESTER_KEYS` before interpolation at nilai lines 41-43, 142, and 206-207. Dynamic placeholder lists are generated by code and all values are bound. The report retains missing-auth findings, not injection findings.
